Современный каталог фильмов на React

MovieCatalog — это полнофункциональное React-приложение для поиска, фильтрации и сохранения информации о фильмах. Проект демонстрирует использование современных веб-технологий для создания отзывчивого пользовательского интерфейса.

Проблемы в современных проектах

Типичные сложности при создании учебных и коммерческих каталогов

Сложность управления состоянием

Без правильной архитектуры состояние приложения (избранное, история поиска) становится сложно синхронизировать между компонентами, что ведет к багам.

Оптимизация производительности

Неоптимизированные запросы к API, отсутствие пагинации и кэширования приводят к медленной работе, особенно при отображении большого количества фильмов.

Адаптивная верстка

Создание интерфейса, который одинаково хорошо выглядит на всех устройствах, требует глубокого знания CSS и тщательного тестирования.

Реализация сложной фильтрации

Разработка удобной системы фильтрации по нескольким критериям (жанр, год, рейтинг) с сохранением состояния и сбросом параметров.

Качество и поддерживаемость кода

Отсутствие TypeScript, единого стиля кода и модульной структуры затрудняет чтение, тестирование и развитие проекта.

Работа с внешним API

Обработка ошибок сети, управление загрузкой, пагинация и обновление данных при изменении параметров запроса.

Технические решения проекта

Как MovieCatalog решает типичные проблемы веб-разработки

Эффективное управление состоянием

Использование Redux Toolkit для централизованного и предсказуемого управления состоянием приложения (избранное, параметры поиска, данные фильмов).

Модульный CSS

Применение CSS Modules для изоляции стилей компонентов, что предотвращает конфликты и упрощает поддержку.

Гибкая система фильтрации

Реализация фильтров с использованием query-параметров URL, что позволяет сохранять и передавать состояние поиска.

Оптимизация запросов

Дебаунсинг поисковых запросов, кэширование ответов API и пагинация для снижения нагрузки и повышения скорости.

Полная типизация

Проект написан на TypeScript, что обеспечивает безопасность типов, автодополнение и сокращает количество runtime-ошибок.

Чистая архитектура

Разделение на компоненты, хуки, сервисы и хранилище, что делает код читаемым, тестируемым и расширяемым.

Ключевые функции

Реализованные возможности в соответствии с учебным заданием

Главная страница с фильмами

Сетка фильмов с постером, названием, рейтингом и годом выпуска. Загрузка данных из The Movie Database (TMDB) API.

Поиск и фильтрация

Поиск по названию и фильтрация по жанрам. Динамическое обновление результатов без перезагрузки страницы.

Избранное

Добавление и удаление фильмов из избранного. Состояние сохраняется между сессиями с использованием локального хранилища.

Детальная страница фильма

Отдельная страница с полной информацией: описание, актерский состав, бюджет, сборы, продолжительность и трейлер.

Адаптивный дизайн

Приложение корректно отображается на устройствах с разным размером экрана благодаря использованию CSS Grid и Flexbox.

Набор тестов

Unit-тесты для критических функций и компонентов с использованием Jest и React Testing Library.

Пользовательские сценарии

Типичные пути взаимодействия с приложением

Пользователь: Анна

Поиск фильма по жанру и добавление в избранное

1

Открытие главной страницы

Анна открывает MovieCatalog. Она видит сетку популярных фильмов, загруженных с TMDB API.

2

Фильтрация по жанру

В сайдбаре она выбирает жанр "Фантастика" и устанавливает минимальный рейтинг 7.0. Список фильмов мгновенно обновляется.

3

Просмотр деталей

Анна кликает на понравившийся фильм, чтобы перейти на его детальную страницу и посмотреть трейлер.

4

Добавление в избранное

На странице фильма она нажимает кнопку "В избранное". Фильм добавляется в её персональную коллекцию.

Киноман: Дмитрий

Создание персональной коллекции и просмотр истории

1

Поиск по ключевому слову

Дмитрий вводит в поисковую строку "космос". Приложение в реальном времени показывает фильмы, соответствующие запросу.

2

Сортировка результатов

Он сортирует результаты по году выпуска (от новых к старым), чтобы увидеть последние релизы.

3

Создание коллекции

Дмитрий добавляет несколько фильмов в избранное, создавая тематическую коллекцию "Космическая фантастика".

4

Просмотр избранного

Он переходит на вкладку "Избранное", где видит все сохранённые фильмы. Состояние сохраняется даже после закрытия браузера.

Этапы разработки

План реализации учебного проекта

Неделя 1-2

Настройка проекта и базовые компоненты

Инициализация React + TypeScript, настройка Redux Toolkit и React Router. Создание основных компонентов: шапка, подвал, карточка фильма.

Неделя 3-4

Работа с API и главная страница

Интеграция с TMDB API, создание сервисов для запросов. Разработка главной страницы с сеткой фильмов и базовой пагинацией.

Неделя 5-6

Система поиска и фильтрации

Реализация поиска по названию и фильтрации по жанрам. Добавление debouncing для поисковых запросов и синхронизация с URL.

Неделя 7-8

Детали фильма и избранное

Создание страницы с детальной информацией о фильме. Реализация функционала избранного с сохранением в localStorage.

Неделя 9-10

Адаптивный дизайн и оптимизация

Адаптивная верстка для мобильных устройств. Оптимизация производительности: lazy loading, кэширование изображений.

Неделя 11-12

Тестирование и документация

Написание unit-тестов для критических компонентов и логики. Создание README с инструкциями по запуску и описанием архитектуры.
